## ¬øPor qu√© otra librer√≠a?

El problema que trata de resolver tRPC es agilizar el desarrollo de aplicaciones teniendo la seguridad que nuestro cliente
y servidor est√°n sincronizados, eso garantiza la consistencia de los datos que se comparten. Es decir, el servidor sabe el tipo
de petici√≥n que le hace el cliente y el cliente sabr√° la estructura de respuesta que el servidor le enviar√°.

Actualmente tenemos tenemos dos formas populares de lograr esto.

### REST + TS

Consiste en agregar tipos a el api desde el servidor y despu√©s compartir esos tipos del lado del cliente o crear tipos espec√≠ficos para el cliente. El procesos ser√≠a algo como lo siguiente:

function getBaseUrl() {
  if (typeof window !== 'undefined') {
    // In the browser, we return a relative URL
    return '';
  }
  // When rendering on the server, we return an absolute URL

  // reference for vercel.com
  if (process.env.VERCEL_URL) {
    return `https://${process.env.VERCEL_URL}`;
  }

  // assume localhost
  return `http://localhost:${process.env.PORT ?? 3000}`;
}

    return {
      links: [
        httpBatchLink({
          url: getBaseUrl() + '/api/trpc',
        }),
      ],

import { initTRPC, TRPCError } from '@trpc/server';
import * as trpcNext from '@trpc/server/adapters/next';
import { z } from 'zod';

type User = { id: number; name: string };
const database: User[] = [
  {
    id: 1,
    name: 'Peter',
  },
  {
    id: 2,
    name: 'Parker',
  },
  {
    id: 3,
    name: 'Spider man',
  },
];

const trpcInstance = initTRPC.create();

const appRouter = trpcInstance.router({
  getClients: trpcInstance.procedure.query(() => {
    return {
      users: [...database],
    };
  }),
  getClient: trpcInstance.procedure
    .input(
      z.object({
        id: z.number().min(0, 'The id needs to be a positive number'),
      })
    )
    .query(({ input }) => {
      const user = database.find((u) => {
        return u.id === input.id;
      });

      if (!user) {
        throw new TRPCError({
          code: 'NOT_FOUND',
          message: 'No user found with that id',
        });
      }

      return {
        user: user,
      };
    }),

  addClient: trpcInstance.procedure
    .input(
      z.object({
        name: z.string(),
      })
    )
    .mutation((ctx) => {
      const { input } = ctx;

      database.push({
        name: input.name,
        id: database.length + 1,
      });
    }),
});

// exportamos la definici√≥n de tipos del API
export type AppRouter = typeof appRouter;

// exportamos API handler para nextjs lo pueda exponer

  router: appRouter,
